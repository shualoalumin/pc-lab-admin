## 현재 구조의 오해 풀기

**학생이 직접 설치할 필요는 없습니다.** Chrome Extension은 관리자(IT 담당자)가 학교 PC에 한 번 설치해두는 것입니다. 학생은 PC를 켜면 이미 설치된 상태로 사용합니다.

하지만 근본적인 질문 — **클라이언트에 아무것도 설치하지 않고 추적이 가능한가** — 에 대한 답은 아래와 같습니다.

---

## 방법별 비교

| 방법 | 클라이언트 설치 필요 | 구현 난이도 | HTTPS 추적 | 현재 구조 연동 |
|------|:-----------------:|:---------:|:---------:|:------------:|
| Chrome Extension (현재) | 필요 (관리자가 설치) | 낮음 | 가능 | 완료 |
| 네트워크 프록시 | 불필요 | 중간 | SSL 인증서 설치 필요 | 가능 |
| DNS 로깅 | 불필요 | 낮음 | 도메인만 추적 (경로 불가) | 가능 |
| Chrome 엔터프라이즈 강제 설치 | 불필요 (GPO 자동 배포) | 높음 (AD/MDM 필요) | 가능 | 기존 Extension 재활용 |
| Windows 에이전트 (서비스) | 필요 (관리자가 설치) | 높음 | 가능 | 가능 |

## 현실적인 대안 두 가지

### 방법 1 — 네트워크 프록시 (클라이언트 완전 무설치)

학교 공유기/스위치 앞단에 프록시 서버를 두는 방식입니다.

```
학생 PC → 프록시 서버 → 인터넷
              ↓
          URL 로그 기록 → Supabase activity_logs
```

- **장점**: 어떤 브라우저를 써도, 어떤 PC를 써도 추적됨
- **단점**: HTTPS는 기본적으로 암호화되어 도메인만 보임 (경로/제목 불가). SSL 인터셉션을 하려면 **학교 CA 인증서를 각 PC에 설치**해야 하므로 결국 PC 설정 작업 필요
- **추천 도구**: Squid Proxy + mitmproxy

### 방법 2 — Chrome 엔터프라이즈 정책으로 자동 배포 (현재 Extension 재활용)

학교가 Windows 도메인(Active Directory)을 운영하고 있다면 GPO(그룹 정책)로 Chrome Extension을 **학생 PC 전체에 자동 강제 설치**할 수 있습니다. 학생이 삭제하거나 비활성화할 수도 없습니다.

```
AD 서버에서 GPO 설정
    ↓
ExtensionInstallForcelist 정책에 Extension ID 추가
    ↓
학교 네트워크 연결 시 Chrome이 자동으로 Extension 설치
```

- **장점**: 현재 만든 Extension 그대로 사용, 학생 개입 완전 불필요
- **단점**: AD 환경이 필요, Chrome Browser Cloud Management 계정 필요

## 결론 및 권장 방향

```
학교 환경에 AD(Active Directory) 있음?
    ├─ YES → GPO로 Extension 자동 배포 (가장 이상적)
    └─ NO
         ├─ 네트워크 관리 가능? → 프록시 서버 구축
         └─ 둘 다 어렵다 → 관리자가 PC마다 수동 설치 (현재 방식)
                          + 학생이 제거 못하도록 Chrome 정책 잠금
```

**현재 구현한 Chrome Extension 방식이 학교 환경에서 가장 현실적입니다.** 관리자가 처음 한 번만 설치하면 되고, 학생 입장에서는 아무 조작이 필요 없습니다. 여기에 GPO로 자동 배포를 추가하면 완전 무개입 운영이 가능합니다.

## RemoteMV가 핵심 보완 레이어

이미 RemoteMV를 운용 중이시다면, 사실 **Extension의 역할을 보조 수단으로 재정의**하는 게 맞습니다.

```
RemoteMV          → 화면 시각 감시 (어떤 우회도 불가)
Chrome Extension  → 자동 URL 로그 + 위반 도메인 자동 차단
```

RemoteMV는 화면 자체를 캡처/스트리밍하기 때문에 시크릿 모드든 Firefox든 의미가 없습니다. 다만 관리자가 실시간으로 쳐다보고 있지 않으면 자동 감지가 안 된다는 게 한계입니다.

## 시크릿 모드 차단 (GPO로 해결 가능)

가장 간단한 해결책은 **Chrome 정책으로 시크릿 모드 자체를 비활성화**하는 것입니다.

```
Windows GPO 또는 레지스트리:
HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Google\Chrome
  IncognitoModeAvailability = 1  (0: 허용, 1: 차단, 2: 강제)
```

이렇게 하면 `Ctrl+Shift+N`을 눌러도 시크릿 창이 열리지 않습니다.

다른 브라우저 차단은 Windows AppLocker나 소프트웨어 제한 정책으로 `.exe` 화이트리스트를 설정하면 됩니다.

## 현실적인 권장 구조

```
[1차 방어] GPO → 시크릿 모드 차단 + Chrome만 허용
    ↓
[2차 방어] Chrome Extension → 자동 URL 로그 + 위반 도메인 즉시 차단
    ↓
[3차 감시] RemoteMV → 화면 시각 감시 (우회 불가능)
    ↓
[4차 기록] Supabase DB → 세션/활동 로그 영구 보관
```

사실 RemoteMV가 있다면 **완전한 우회는 불가능**합니다. Extension은 반복적인 위반을 자동으로 감지/기록/차단하는 자동화 레이어이고, RemoteMV는 사람이 확인하는 최종 방어선입니다. 두 레이어가 서로를 보완하는 구조입니다.
